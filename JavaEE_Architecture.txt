

자바 EE 애플리케이션의 아키텍처


1. 클라이언트 티어

   클라이언트 티어는 클라이언트-서버 구조에서 클라이언트에 해당하는 애플리케이션을 전부 일컫는 말.
   그러나 HTTP로 자바EE서버에 접속하는 브라우저가 대부분


2. 미들티어

   2-1. 웹 레이어
      웹 레이어는 클라이언트 티어와 비즈니스 레이어 간 상호 작용을 관장합니다.
      클라이언트 티어는 웹 레이어에게 리소스를 요청한다. 요청 정보에는 ID/PW 등 사용자 입력 데이터나 로그인 정보가 담겨있다.
      웹 레이어는 세션에 사용자 상태를 담아두고 부분적으로 비즈니스 로직을 수행하거나 메모리에 임시로 데이터를 담아놓습니다.
      그리고 웹 레이어의 기술은 클라이언트 티어와 미들티어 간 움직음을 관장하고 응답을 생성하는 문제에 관한 것.
 
   2-2. 비즈니스 레이어
      비즈니스 레이어는 업무 요건을 처리하거나 도메인 내부의 특정 비스니스 로직을 실행합니다.
      그 과정에서 EIS 티어에 있는 DB에서 데이터를 가져오고 클라이언트 데이터를 취합함.
      비즈니스 로직은 EJB에 보관되어 있고 사용할 데이터는 JAVA Persistance API, JAVA Transaction API, JAVA Database Connectivity,      JDBC 기술로 EIS에서 가져옴.
